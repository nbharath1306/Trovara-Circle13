# Project Structure

## Full Folder Layout

```
trovara/          ← Root folder (git repo)
│
├── manage.py                    ← Django management commands
├── requirements.txt             ← All Python dependencies
├── .env                         ← Secret keys (never commit this)
├── .env.example                 ← Template showing required variables
├── .gitignore
├── Procfile                     ← Tells Render how to run the app
├── render.yaml                  ← Render deployment config (optional)
│
├── trovara/              ← Django project settings package
│   ├── __init__.py              ← Imports Celery app (important!)
│   ├── settings.py              ← All Django settings
│   ├── urls.py                  ← Root URL config
│   ├── celery.py                ← Celery app configuration
│   └── wsgi.py                  ← WSGI entry point for Gunicorn
│
├── agent/                       ← Main Django app
│   ├── __init__.py
│   ├── models.py                ← ResearchTask model
│   ├── serializers.py           ← DRF serializers
│   ├── views.py                 ← DRF API views
│   ├── urls.py                  ← App-level URL patterns
│   ├── admin.py                 ← Django admin registration
│   ├── apps.py                  ← App config
│   ├── tasks.py                 ← Celery agent task (the brain)
│   │
│   ├── services/                ← Agent tool functions
│   │   ├── __init__.py
│   │   ├── search.py            ← DuckDuckGo web search
│   │   ├── scraper.py           ← requests + BeautifulSoup page reader
│   │   └── llm.py               ← Groq API (summarize + report)
│   │
│   └── migrations/              ← Auto-generated by Django
│       ├── __init__.py
│       └── 0001_initial.py
│
└── templates/
    └── index.html               ← Single-page frontend dashboard
```

## Key File Descriptions

### `trovara/celery.py`
Initializes the Celery application and connects it to Django settings. Must be imported in `__init__.py`.

### `trovara/__init__.py`
```python
from .celery import app as celery_app
__all__ = ("celery_app",)
```
This makes Celery load when Django starts.

### `agent/tasks.py`
The autonomous agent. A single Celery task that runs the full 4-step pipeline. This is the most important file in the project.

### `agent/services/`
Clean separation of concerns — each tool the agent uses is its own module:
- `search.py` = the search tool
- `scraper.py` = the web reading tool
- `llm.py` = the summarize and generate tools

### `templates/index.html`
A single HTML file served by Django. Uses JavaScript `fetch()` to call the REST API and display live results. No React needed.

### `Procfile`
```
web: gunicorn trovara.wsgi --log-file -
worker: celery -A trovara worker --loglevel=info
```
Render reads this to know it needs to run two processes: the web server and the Celery worker.
